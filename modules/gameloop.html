fixint (updatehz)
Sets up a game loop with a fixed timestep and interpolated rendering. Replaces the default love.run callback with one that calls love.update updatehz times per second. Additionally, love.draw will receive an argument alpha that tells it how much it needs to interpolate the scene by for this frame. That is, objects in a scene need to be drawn at a position that is alpha percent between the previous and next update frame. This game loop aims to eliminate a visual artifact known as temporal aliasing when used properly.

love.timer is required for this game loop to function. If it is disabled, love.run will throw an error.

Parameters:

  • updatehz int How many times to update the game per second.
